ベイズの定理 — 証拠で信念を更新する

$$P(\theta\mid D) = \frac{P(D\mid\theta)\,P(\theta)}{P(D)} \qquad \text{すなわち}\quad \text{事後分布} \propto \text{尤度}\times\text{事前分布}$$

事前分布 P(θ)(データを見る前の信念)に、尤度 P(D|θ)(そのθでこのデータが出る度合い)を掛けて規格化したものが事後分布 P(θ|D)。「事前×尤度→事後」の1ステップがベイズ推論のすべてで、データが増えるたびにこの更新を繰り返せる(昨日の事後が今日の事前になる)。

有病率1%の病気の検査。感度(病気の人が陽性になる確率)90%、特異度(健康な人が陰性になる確率)95%。陽性と出た人が本当に病気である確率は?
解答を見る

P(病|陽) = P(陽|病)P(病) / [P(陽|病)P(病) + P(陽|健)P(健)]
= 0.9×0.01 / (0.9×0.01 + 0.05×0.99) = 0.009/0.0585 ≈ 0.154(約15%)
直感(90%!)と大きくずれるのは、事前確率(有病率1%)が低いから。ベイズの定理は「検査の性能」と「そもそもの起こりやすさ」を正しく合成する。

共役事前分布 — 更新が閉じた式で回る組み合わせ

なぜ学ぶのか: ベイズ更新は一般には「事前×尤度」の積分計算が必要で、手計算ではほぼ不可能。ところが事前分布をうまく選ぶと、事後分布が同じ族の分布になり、更新がパラメータの足し算だけで済む — 積分が消えて暗算レベルになる。これが共役性で、①手計算できる唯一の実用ルートとして準1級の計算問題の定番、②データが来るたびに逐次更新できる(昨日の事後=今日の事前)、③MCMC(数値計算)が要らないベースラインとして実務でも現役。代表例:

$$\text{事前 } \mathrm{Beta}(\alpha,\beta) \;\times\; \text{尤度 二項}(s\ \text{成功},\, f\ \text{失敗}) \;\longrightarrow\; \text{事後 } \mathrm{Beta}(\alpha+s,\, \beta+f)$$

ベータ分布のパラメータは「仮想的な成功数 α・失敗数 β」と読める。つまり事前分布とは「データを見る前に α+β 回分の経験を持っている」ことに相当し、実データ s, f がそこに合流していく。事後平均は (α+s)/(α+β+s+f) — 事前の重みとデータの重みの加重平均になっている。

ベイズ更新ラボ(ベータ×二項)— 事前×尤度→事後 が動く様子を見る

試してほしい操作:まず s=f=0 にする → 事後=事前(データがなければ信念は変わらない)。s を増やす → 事後(青実線)が右へ移動し尖っていく。事前を Beta(1,1)(一様)にする → 事後が尤度(橙点線)と一致 — 「事前知識ゼロならデータが全て」。逆に事前を α=30, β=30 と強くしてデータを少しにする → 事後が事前からほぼ動かない(頑固な事前)。データを増やすほど事前の影響が薄れることを、s, f を大きくして確認。

🔧 実装メモ(このラボが実際に計算していること)
  • ベータ分布pdf: f(x)=x^(α−1)(1−x)^(β−1)/B(α,β)。対数領域で計算し、ln B(α,β)=lnΓ(α)+lnΓ(β)−lnΓ(α+β) は共通エンジンの lnGamma(Lanczos近似)を使用
  • 3本の曲線: 事前 Beta(α,β)(灰点線)/ 正規化した尤度 = Beta(s+1, f+1)(橙点線)/ 事後 Beta(α+s, β+f)(青実線)。尤度は面積1に規格化して同じ土俵で表示
  • 読み出し値: 事後平均 (α+s)/(α+β+s+f)、MAP (α+s−1)/(α+β+s+f−2)(分母>0のとき)。95%信用区間は数値積分(累積台形則、格子501点)の2.5%・97.5%分位点による近似
  • 軸固定: x∈[0,1]、y上限12(α+s が大きく尖った場合ははみ出しをクリップ — 上端で切れていたら「それだけ確信が強い」ということ)
  • α,β<1(端で発散する形状)はスライダー範囲外(最小0.5では発散するがクリップで対応)。Jeffreys事前 Beta(0.5,0.5) は選択可能
頻出ポイント:①事後平均 (α+s)/(α+β+s+f) は「事前平均 α/(α+β)」と「標本比率 s/(s+f)」の加重平均に分解できる(重みは α+β と s+f)。②一様事前 Beta(1,1) のときの MAP は最尤推定量 s/(s+f) に一致する。③正規(分散既知)×正規、ポアソン×ガンマも同じ構造(事後精度=事前精度+データ精度)。

MCMC — 事後分布から「歩いて」サンプルする(考え方だけ)

共役でない一般のモデルでは事後分布の規格化定数 P(D) が計算できない。そこで「事後分布を定常分布にもつマルコフ連鎖を設計し、その連鎖を長く走らせてサンプルを集める」のがMCMC(メトロポリス法・ギブスサンプリング)。比 P(θ′|D)/P(θ|D) には P(D) が現れない — これが「規格化定数いらず」の種。次節のマルコフ連鎖の収束(定常分布への到達)が、まさにMCMCが機能する理論的根拠になる。

マルコフ連鎖 — 「直前だけが未来を決める」

$$\text{マルコフ性: } P(X_{t+1} \mid X_t, X_{t-1}, \dots) = P(X_{t+1} \mid X_t)$$

状態の移り変わりを遷移行列 P(行=今の状態、列=次の状態、各行の和=1)で表す。時刻 t の分布を行ベクトル πₜ とすると πₜ₊₁ = πₜP。定常分布 π は πP = π を満たす分布で、既約(全状態が行き来可能)かつ非周期なら、どんな初期分布から出発しても πₜ → π に収束する(エルゴード性)。

$$\text{2状態の例: } P(\text{晴}\to\text{雨})=p,\; P(\text{雨}\to\text{晴})=q \;\text{のとき}\quad \pi_{\text{晴}} = \frac{q}{p+q}$$
マルコフ連鎖ラボ — どこから出発しても同じ分布に吸い込まれる

試してほしい操作:3本の曲線は「初日晴れ(P₀(晴)=1)」「初日雨(=0)」「半々(=0.5)」から出発した P(晴) の時間発展。出発点が違っても全員、緑の点線(定常分布)へ収束する — これがエルゴード性であり、MCMCで「バーンイン後は初期値を忘れる」と言われる根拠。p, q を両方大きく(0.95)する → 交互に行き来して振動しながら収束(固有値 1−p−q が負)。p, q を両方小さく → 収束が遅い(状態に粘る)。p と q の比率を変える → 定常分布の高さ q/(p+q) が動く。

🔧 実装メモ(このラボが実際に計算していること)
  • モデル: 2状態マルコフ連鎖。遷移行列 P=[[1−p, p],[q, 1−q]]。分布の時間発展は πₜ₊₁=πₜP の厳密な行列反復(t=0…30)で、乱数シミュレーションではない
  • 定常分布: π晴=q/(p+q) の閉じた式(緑点線)。収束の速さは第2固有値 λ₂=1−p−q の絶対値で決まり、readoutに表示
  • 3本の曲線: 初期分布 π₀(晴)=1, 0, 0.5 の3通りを同時に描画(初期値によらない収束を見せるため)
  • 軸固定: t∈[0,30]、y∈[0,1]
  • 3状態以上・周期的連鎖(収束しない例)・吸収マルコフ連鎖・実際のMCMCサンプリングは未実装
遷移確率 P(晴→雨)=0.3, P(雨→晴)=0.6 のマルコフ連鎖の定常分布を求めよ。また今日が雨のとき、十分先の日が晴れである確率は?
解答を見る

πP=π より π晴×0.3 = π雨×0.6(流出=流入のバランス)。π晴+π雨=1 と合わせて
π晴 = 0.6/(0.3+0.6) = 2/3, π雨 = 1/3。
エルゴード性により初期状態は忘れられるので、十分先が晴れの確率も 2/3(上のラボでp=0.3, q=0.6にすると全曲線が2/3≈0.667に収束するのが見える)。
