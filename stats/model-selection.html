モデル選択:AIC・交差検証・正則化

タグ: AIC / BIC / 情報量規準 / 交差検証 / クロスバリデーション / 正則化 / リッジ回帰 / Lasso / 過学習

フェーズ2対応済み: 次数選択(AIC×交差検証)ラボつき。

前提知識: 過学習とバイアス・バリアンスの直感は回帰分析のラボで先に体感を。尤度・最尤法は推定にあります(AICの式に出てくる L̂ はそれ)。

全体マップ — この分野の体系

「当てはまりを良くするだけなら複雑なモデルが必ず勝つ」— 回帰分析で見た過学習の問題への3つの解答が、①罰則付きの当てはまり指標(情報量規準)、②未知データで直接測る(交差検証)、③そもそも複雑になれないよう縛る(正則化)。

  • モデル選択(複雑さと当てはまりのバランスを取る)
    • 情報量規準 — 尤度にパラメータ数の罰則を課す
      • AIC — 予測の良さを狙う
      • BIC — 真のモデルの特定を狙う(罰則がより重い)
      • 自由度調整済みR²(回帰ページ)・Cp・DIC — 触りのみ
    • 交差検証(CV)
      • k-fold CV・LOOCV — データを分けて予測性能を直接測る
    • 正則化(縮小推定)
      • リッジ回帰(L2罰則)・Lasso(L1罰則)
方法式・手順狙い特徴
AIC−2 ln L + 2k予測誤差の最小化nが大きくても比較的複雑なモデルを許す
BIC−2 ln L + k ln n真のモデルの選択(一致性)n≧8で罰則がAICより重く、より単純を選ぶ
k-fold CVk分割し「残り1つで検証」を一巡予測誤差の直接推定仮定が少なく万能。計算コスト大
リッジ/Lasso残差二乗和+λΣβ²/λΣ|β|分散を抑えて予測改善Lassoは係数をちょうど0にできる(変数選択)

AIC — 「当てはまり − 複雑さ」の採点

なぜ学ぶのか: モデル候補が複数あるとき「どれを使うべきか」を決める客観的な物差しが要る。当てはまり(R²や尤度)だけで選ぶと必ず複雑なモデルが勝ってしまい過学習する — そこで複雑さの罰金込みで採点し、モデル選択を1つの数字の比較に落とすのが情報量規準。準1級では「AICを計算してモデルを選べ」という直接の出題がある。

$$\mathrm{AIC} = -2\ln\hat{L} + 2k \qquad (\hat{L}:\ \text{最大尤度},\; k:\ \text{パラメータ数})\text{。小さいほど良い}$$

第1項は当てはまりの良さ(小さいほど良い)、第2項が複雑さの罰金。理論的には「予測分布と真の分布のKL距離の推定量」で、予測を良くするための規準。絶対値に意味はなく同じデータでのモデル間の差だけが意味を持つ(目安: 差が2未満なら実質互角)。BICは罰則が k ln n で、n が大きいほど単純なモデルへ傾く — 「予測のAIC、真実のBIC」という目的の違いで使い分ける。

交差検証 — 「見せていないデータ」で採点する

データをk個(標準は5か10)に分割し、k−1個で学習して残り1個で予測誤差を測る、をk回転して平均する。訓練誤差(見せたデータでの成績)は常に楽観的なので、未知データでの成績を推定するのが目的。k=n の特別な場合がLOOCV(1個抜き)。回帰分析ページのバンド幅・次数などのチューニングパラメータ選択の標準手段。

CVの頻出注意:①前処理(標準化・変数選択)も各foldの学習側だけで行うこと — 全データでやってから分割すると情報漏れ(リーケージ)で楽観バイアス。②時系列データでは未来を訓練に使わないよう時間順の分割を使う。

正則化 — 係数に「重り」を付けて暴れを防ぐ

$$\text{リッジ: } \min_\beta \sum(y_i-\mathbf{x}_i\beta)^2 + \lambda\sum_j \beta_j^2 \qquad \text{Lasso: } \min_\beta \sum(y_i-\mathbf{x}_i\beta)^2 + \lambda\sum_j |\beta_j|$$

λ が複雑さのダイヤル(λ=0で通常の最小二乗、λ→∞で全係数0)。少しのバイアスと引き換えに分散を大きく減らすのが狙いで、多重共線性や「変数がデータ数より多い」状況で威力を発揮する。L1罰則(Lasso)は解の角の性質により一部の係数を厳密に0にする=自動変数選択ができる点がリッジとの本質的な違い。λはCVで選ぶのが標準。

ベイズ的な見方: リッジは係数に正規事前分布、Lassoはラプラス事前分布を置いたMAP推定と等価。ベイズ法の「事前分布が推定を安定させる」という話と同じ構造が、ここでは「罰則」の顔で現れている。
モデルA: パラメータ3個で ln L̂=−50、モデルB: パラメータ6個で ln L̂=−47。AICではどちらを選ぶか。n=100のときBICでは?
解答を見る

AIC_A=100+6=106, AIC_B=94+12=106 → AICでは互角(差0)。
BIC(ln 100≈4.605): BIC_A=100+13.8=113.8, BIC_B=94+27.6=121.6 → BICはAを選ぶ。
罰則の重いBICが単純なモデルに傾く典型例。

リッジ回帰とLassoの共通点と、決定的に違う点をそれぞれ1つ挙げよ。
解答を見る

共通点: 係数を0方向に縮小してバイアスを許容し分散を減らす(λはCVで選択)。
相違点: Lasso(L1)は係数をちょうど0にできるので変数選択を兼ねる。リッジ(L2)は0に近づけるだけで0にはしない(多重共線性の下でも全変数を残して安定化)。
